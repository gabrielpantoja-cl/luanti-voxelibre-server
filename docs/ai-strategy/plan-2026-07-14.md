## Format translation map (Claude Code → OpenCode)

| Claude Code | OpenCode |
|---|---|
| Agent `model: sonnet` | Drop. OpenCode inherits global `opencode.json` default = `MiniMax/MiniMax-M3` |
| Agent body (Spanish prompt) | Same body content under `.opencode/agents/<name>.md` (filename = agent name) |
| Command `allowed-tools: Bash(git *)` | Drop. Command uses `agent:` frontmatter; that agent's `permission` map applies |
| Command `argument-hint` | Drop. OpenCode uses `$ARGUMENTS` / `$1..$N` in the body |
| Skill `disable-model-invocation: true` | Drop. OpenCode skills load on demand via the `skill` tool by name (same effective behavior) |
| Skill `allowed-tools: Bash(magick *)` | Drop (see above) |
| Rule `globs: server/mods/**/*.lua` | Drop. OpenCode skills discover by `description`, not globs (better UX) |
| Per-file frontmatter unknown fields | OpenCode silently ignores them (per v1.1+ docs) |

**Net effect**: every migrated file gets its frontmatter trimmed to OpenCode-recognized fields only (`name`, `description`, `mode`, `agent`, `temperature`, etc.). Body content is preserved verbatim.

### Phase 5 — Medium/high risk (gated by user approval)

#### 5a. Pre-flight test of junction/symlink in `/tmp`

**Mandatory before any production shim.**

```bash
# Linux/macOS
mkdir -p /tmp/oc-shim-test && cd /tmp/oc-shim-test
mkdir real
echo "hello" > real/file.md
ln -s "$(pwd)/real" link
ls -la link  # should list file.md through the link
cat link/file.md  # should print "hello"

# Windows PowerShell
$test = "$env:TEMP\oc-shim-test"
New-Item -ItemType Directory -Path "$test\real" -Force | Out-Null
Set-Content -LiteralPath "$test\real\file.md" -Value "hello"
New-Item -ItemType Junction -Path "$test\link" -Target "$test\real"
Get-Content "$test\link\file.md"  # should print "hello"
```

Validation: link resolves to target; reading through the link returns the file content. **Must succeed on the target host OS before any work on the real repo.** Linux symlinks are first-class POSIX objects; Windows junctions are NTFS reparse points and have slightly different semantics (no network paths, no relative targets).

Pre-conditions: `mkdir`/`New-Item` access to `/tmp` or `%TEMP%`. **Risk: none** (temp dir).
Rollback: `rm -rf $test` (no production impact).

#### 5b. Portable setup script `scripts/setup-ai-shims.{sh,ps1}`

**`scripts/setup-ai-shims.sh`** (Linux/macOS, primary):
- Detects `$OSTYPE`
- For each shim pair (`.claude/agents` ↔ `.opencode/agents`, `.claude/commands` ↔ `.opencode/commands`, `.claude/skills` ↔ `.opencode/skills`):
  - Verifies `.opencode/<dir>` exists and is populated
  - Verifies `.claude/<dir>` exists and is not already a symlink
  - Backs up `.claude/<dir>` to `.claude/<dir>.bak-$(date +%s)` (only if not already backed up)
  - Removes the original `.claude/<dir>` (only after backup is confirmed)
  - Creates `ln -s "../.opencode/<dir>" ".claude/<dir>"` (relative path)
- Verifies all shims resolve
- Prints summary

**`scripts/setup-ai-shims.ps1`** (Windows, secondary):
- Same logic using `New-Item -ItemType Junction` instead of `ln -s`
- Backup uses `Copy-Item -Recurse`
- Prints summary

Pre-conditions: 5a pre-flight passed on this OS. **Risk: low** (script creates backups).
Validation: dry-run via `WHATIF=1 scripts/setup-ai-shims.sh` or `-WhatIf` parameter.
Rollback: `rm` the symlink, `mv` the backup back.
