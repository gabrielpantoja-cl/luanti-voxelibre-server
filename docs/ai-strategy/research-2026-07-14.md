## 3. Current best-practice config schema

### `opencode.json` (canonical root config — JSON or JSONC)
```
{
  "$schema": "https://opencode.ai/config.json",
  "model": "provider/model-id",
  "small_model": "provider/model-id",
  "permission": { ... },            // default-deny explicitly when needed
  "provider": { ... },              // timeout, chunkTimeout, setCacheKey
  "agent": { ... },                 // primary + subagent definitions
  "command": { ... },               // also configurable as JSON (alt to .md)
  "instructions": ["..."],          // auto-loaded instruction files (very useful!)
  "enabled_providers": ["..."],     // whitelist (overrides default 75+)
  "disabled_providers": ["..."],    // blacklist (priority over enabled)
  "snapshot": true,                 // file tracking for /undo
  "compaction": { "auto": true, "prune": false, "reserved": 10000 },
  "attachment": { "image": { ... } },
  "experimental": { "policies": [...] }
}
```

### Skills discovery paths
- Project: `.opencode/skills/<name>/SKILL.md`
- Project Claude-compatible: `.claude/skills/<name>/SKILL.md` and `.agents/skills/<name>/SKILL.md`
- Global: `~/.config/opencode/skills/`, `~/.claude/skills/`, `~/.agents/skills/`

### Agents
- JSON inline in `opencode.json` under `agent.<name>` OR markdown at `.opencode/agents/<name>.md` (filename = agent name)
- Per-agent permissions override global
- Built-in agents: `build`, `plan` (primary); `general`, `explore`, `scout` (subagents). Hidden: `compaction`, `title`, `summary`.

### Commands
- Markdown at `.opencode/commands/<name>.md`
- Supports `$ARGUMENTS`, `$1/$2/$3`, `!shell`, `@file` in body
- Per-command `agent:`, `model:`, `subtask:` overrides

### Permissions
- Three actions: `allow` / `ask` / `deny`
- Default-deny only what you want blocked; rest stays permissive (`allow`)
- Per-tool object syntax with glob patterns (e.g. `"git *": "allow"`)
- **Last matching rule wins** — order matters
- `.env` files default to `deny` for `read`

## 4. Provider & model notes

### Primary model: MiniMax-M3
- **Provider ID in OpenCode**: `MiniMax` (built-in; one of 75+ providers)
- **Model**: MiniMax-M3 — *Frontier multimodal coding model with 1M context window*
- **Capabilities** (per https://platform.minimax.io/): multimodal, 1M context window, frontier coding
- **Sign-in path**: `MiniMax API Console` → `/connect` in TUI → search `MiniMax` → paste API key → `/models` to pick `M3` (current) rather than the documented `M2.1`
- **Data residency**: Per OpenCode provider docs, MiniMax routes through the MiniMax API platform. Treat as a **paid third-party hosted provider** — data sensitivity should be ≤ Tier C (public/aggregated/code-level) for safe use. This repo is **Tier D (code & tooling)** so it's fully compatible.
- **Pricing / Token Plan**: User is on a Token Plan (pay-per-token). No fallback configured — MiniMax-M3 is the only model in use.

### Why 1M context changes the calculus
- The whole Wetlands repo (excluding `server/worlds/*.sqlite`) is **well under 1M tokens**. A single session can absorb the entire `AGENTS.md` (28KB), `CLAUDE.local.md` (6KB), all `docs/00-SHARED/**/*.md`, all `server/mods/**/*.lua` configs, the full game-mod inventory in one shot.
- Implication: **manual context curation matters less**; the main reason to add `instructions: [...]` is **auto-loading on session start**, not "fitting it in".

### Cache & timeout tuning
- For long sessions set `compaction.prune: true` to drop old tool outputs.
- `provider.MiniMax.options.timeout` defaults to 300000 (5 min); bumping to 600000 (10 min) is reasonable for `map.sqlite` queries.
